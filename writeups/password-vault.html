Secure Password Vault

Project: AES-256-GCM Password Vault
Author: ZIYAN

Why password storage is dangerous

Most cyber attacks begin with stolen or reused credentials. If one site leaks a password, attackers try the same password on email, banking, and cloud accounts. This is called credential stuffing.

How the vault protects credentials

The vault never stores your master password. Instead, it uses PBKDF2 to derive a strong encryption key from it. All saved passwords are then encrypted using AES-256-GCM, which provides both confidentiality and tamper detection.

Why AES-256-GCM matters

AES-GCM ensures that encrypted data cannot be modified without being detected. This protects the vault from silent corruption, malware tampering, and data injection attacks.

What an attacker sees

If an attacker steals the vault file, they see only random encrypted bytes. Without the master password, decrypting the data is computationally infeasible.

Security impact

This vault prevents credential theft, reduces breach damage, and enforces proper cryptographic storage of secrets. It reflects how professional password managers work internally.
